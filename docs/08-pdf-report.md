# PDF Report

> The monthly artifact that lands in a parent's email. Where club branding most visibly shines, and where the session summary field's value is most fully realized.

## Purpose

The PDF report is the **most important external artifact** PadelCoach produces. Parents and trainees take it out of the app and into their lives — print it, share it, archive it, refer back to it. It's the artifact that justifies the subscription cost in the parent's eyes.

Generated automatically on the 1st of each month, or manually on demand by a coach.

## Audience

- **Primary:** Parent of junior trainee (paying customer)
- **Secondary:** Adult trainee (reads it themselves)
- **Tertiary:** Coach (reviews before sending, occasionally archives for records)

## Format

- **Page size:** A4 portrait (210 × 297 mm) — Indonesia standard. International deliveries can use US Letter via setting (V1.5).
- **Length:** 1 page typical. 2 pages if session log is long (>5 entries).
- **Output:** PDF (primary) + permanent web URL `padelcoach.app/r/{trainee-slug}-{period-slug}` (canonical record).

## Generation strategy

Two paths:

### Auto-generated (default — 1st of each month)
1. Aggregate the period data
2. Concatenate session summaries written by the coach during the month
3. Use Claude to compose the monthly "Coach's note" from those summaries (preserving voice, removing repetition)
4. Compute "May goals" from current tier blockers + coach defaults
5. Render to PDF, store, and email to parent contact + trainee
6. Log a notification visible to coach: "April report sent — review and edit if needed"

### Manual (coach taps "Create report" anytime)
Same pipeline but triggered ad-hoc, with the coach reviewing and editing the auto-drafted note before sending.

### Coach edit before send

Even auto-generated reports stage as drafts. Coaches can override:
- The monthly "Coach's note"
- The "May goals" list
- Which sessions are included in the log (default: all)

Once approved, the report is locked, gets a permanent URL, and is sent.

## Per-club rendering

The report is **fully themed** per workspace. Single accent CSS variable swap propagates to:
- Brand mark (left of letterhead)
- Tier pills
- Progress bar fill
- Section accent backgrounds (Coach's note card, Looking ahead card)
- Radar polygon fill
- Win icons (uses success green, not accent)
- Bullet markers in goals list

Logo image renders if uploaded; falls back to initials.

## Web URL fallback

Every PDF has a corresponding web page at `padelcoach.app/r/{slug}`. Same content, responsive layout, embedded in a clean web frame. This serves:
- Parents who want to view on a different device without re-downloading
- Coaches sharing the report URL in messages
- Long-term archive (PDF locally lost? URL persists)
- SEO-discoverable (with public-share toggle, V1.5)

## States

### Default (sent)
PDF + URL live.

### Draft (auto-generated, awaiting coach review)
Coach sees a "Review draft" banner on Reports tab. Tapping opens an editable preview with [Send] and [Discard] actions.

### Empty (no data in period)
"Not enough activity to generate a report" — only shown if 0 sessions in the period. Skipping period entirely is also valid.

### Failed generation
Error logged. Coach gets notification: "Couldn't generate April report · Tap to retry". Doesn't auto-resend.

## Design rationale

**Why the Coach's note is the visual hero, not the metrics.** Parents read sentences, not radars. Leading with narrative prose set in a tinted accent-bg card creates the right hierarchy: human voice first, supporting evidence second. Most fitness/coaching reports vomit data and bury the insight. We invert that.

**Why session log preserves the coach's actual summary text.** This is where the in-app session summary field most clearly pays off. Without it, the session log is metadata. With it, the parent reads a coaching narrative — "Andi struggled reading opponent's lobs, stayed flat-footed" — that justifies the monthly fee far better than a chart could.

**Why the stats grid is 2×2, not 4-across.** Portrait PDF is narrow; 4 thin numbers cramp each other. Two-up gives each stat enough breathing room. The "100% attendance rate" is a deliberate parent-facing flex — parents care about reliability as much as skill progression.

**Why "Looking ahead" uses bulleted prose, not a checklist.** Checklists imply commitment ("did we complete this goal?"); bullets imply intent ("here's where we're aiming"). For a coaching report, the latter is more honest — coaches set direction, but the actual work happens session by session.

**Why the signoff line at the end ("Excited to see where Andi takes this").** Forward-looking emotional beat is structurally important. The report ends in possibility, not in summary. Parents close the PDF feeling like next month is already promising.

**Why a permanent URL exists for every PDF.** Free architecture win — once the report renderer exists, hosting it as a web page costs nothing. Parents who lose the PDF still have the URL. Coaches sharing the URL in messages avoids re-downloading. Long-term archive is implicit.

**Why we don't show comparison to other trainees ("ranked 3rd in your tier").** Competitive ranking is corrosive in a coaching context, especially for kids. The whole product philosophy is per-trainee progress, not relative competition. Parents asking "how is my kid doing vs the others?" should hear the answer from the coach in person, not from a chart.

**Why we don't render every assessment detail.** Per-skill scores are in-app. The PDF is the *summary* artifact — a few headlines, the narrative, the highlights. Adding a 27-row skill table bloats the PDF and dilutes the narrative.

**Why "Generated by PadelCoach" footer.** Trust signal + product attribution. Parents who like the report know what tool produced it; if they coach kids elsewhere, they may suggest the product to other coaches. Soft growth loop.

## Out of MVP scope

## Open questions

1. Auto-generation timing: 1st of month, or last day of previous month? **Recommendation:** 1st of month at workspace's local time zone — parents check email in the morning, get a digest of what just ended.
2. Email subject line format? **Recommendation:** "Andi Pratama — April progress · Senayan Padel Club"
3. Email body — full report inline or just link? **Recommendation:** brief teaser ("Here's Andi's April report. Coach Novia: 'Strong month — forehand reached Functional.'") + PDF attachment + link to web version.
4. Expiry / privacy on the web URL — public or login-required? **Recommendation:** unguessable slug, public read (parents share with grandparents), can be revoked from settings (V1.5).
5. If a coach edits a sent report, do we re-send or version it? **Recommendation:** v1 sent is locked; edits create a v2 marked as "Updated" — parent gets a notification and the URL serves the latest.
